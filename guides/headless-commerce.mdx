---
title: "Build a Custom Checkout with Fluid's Commerce API"
sidebarTitle: "Headless Commerce"
description: "Build a fully custom checkout experience for your direct sales organization using Fluid's headless Commerce API, from product listing to order completion."
---

Fluid's Headless Commerce API gives you complete control over the buyer experience — without being locked into a default storefront. You call the API directly to list products, manage carts, and complete orders, while Fluid handles the underlying logic specific to direct sales organizations: downline placement, distributor enrollments, pricing tiers, and more. This guide walks you through the full checkout lifecycle from authentication to order confirmation.

<Steps>

  <Step title="Authenticate">

    Before making any Commerce API request, obtain your API credentials from the Fluid dashboard. Fluid supports two credential types for server-side use:

    - **Company tokens** — tied directly to your organization
    - **Partner tokens** — recommended for third-party integrations; can be managed programmatically and support expiration dates

    Use your company or partner token to authenticate product listing and cart creation requests. These credentials should **never** be exposed in client-side code.

    Once you create a cart (Step 3), you receive a **cart token** scoped to that single cart instance. Use the cart token for all subsequent checkout operations from the browser or mobile client — it is safe to use on the client side.

    For full details on obtaining and rotating credentials, see the [Authentication Guide](/guides/authentication).

    <Note>
      The two-tier authentication model lets you keep your API credentials secure on the server while still powering rich client-side checkout experiences via the scoped cart token.
    </Note>

  </Step>

  <Step title="List Products">

    Use the Product API to fetch your organization's catalog and retrieve a `variant_id` to add to the cart.

    ```bash
    GET https://{subdomain}.fluid.app/api/company/v1/products
    ```

    The endpoint supports filtering, pagination, and search, so you can build tailored browsing experiences:

    - Filter by category, availability, or custom attributes
    - Paginate large catalogs with `page` and `per_page` parameters
    - Search by keyword or SKU

    <CodeGroup>

    ```bash cURL
    curl -X GET "https://myco.fluid.app/api/company/v1/products" \
      -H "Authorization: Bearer YOUR_COMPANY_TOKEN" \
      -H "Content-Type: application/json"
    ```

    ```javascript JavaScript
    const response = await fetch(
      'https://myco.fluid.app/api/company/v1/products',
      {
        headers: {
          Authorization: 'Bearer YOUR_COMPANY_TOKEN',
          'Content-Type': 'application/json',
        },
      }
    )
    const { products } = await response.json()
    ```

    </CodeGroup>

    The response includes each product's title, price, description, and variants (with per-country pricing). Note the `variant_id` of the item you want to add to a cart — you'll need it in the next step.

    For full filtering parameters and response schemas, see the [Catalog API Reference](/docs/openapi/catalog-v1).

  </Step>

  <Step title="Create a Cart">

    POST to the cart endpoint with the `variant_id` from the previous step to start a checkout session. The API returns a secure **cart token** you will use for all remaining steps.

    <CodeGroup>

    ```bash cURL
    curl -X POST "https://myco.fluid.app/api/carts" \
      -H "Authorization: Bearer YOUR_COMPANY_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{
        "cart": {
          "variant_id": 8841,
          "quantity": 1
        }
      }'
    ```

    ```javascript JavaScript
    const response = await fetch('https://myco.fluid.app/api/carts', {
      method: 'POST',
      headers: {
        Authorization: 'Bearer YOUR_COMPANY_TOKEN',
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        cart: {
          variant_id: 8841,
          quantity: 1,
        },
      }),
    })
    const { cart } = await response.json()
    const cartToken = cart.token // e.g. "CT-SECURE_TOKEN_XYZ"
    ```

    </CodeGroup>

    The response includes a `token` field — for example `CT-SECURE_TOKEN_XYZ`. Store this value in your application state; every subsequent cart operation requires it.

    <Note>
      The cart token is scoped to a single cart instance and is safe to use directly from a browser or mobile app. It does not grant access to your broader API credentials.
    </Note>

    For additional cart creation options (bulk item addition, special attributes), see the [Cart API Reference](/docs/openapi/carts-v0).

  </Step>

  <Step title="Update the Cart">

    With the cart token in hand, you can update the cart in any order before checkout. Each update returns the full current cart state, including recalculated taxes, shipping costs, and totals.

    **Add or remove items**

    ```bash
    POST https://myco.fluid.app/api/carts/{cart_token}/items
    ```

    Provide a `variant_id` and `quantity` for each item. To remove an item, set its quantity to `0` or issue a DELETE to the item endpoint.

    **Set the customer email**

    ```bash
    PATCH https://myco.fluid.app/api/carts/{cart_token}
    ```

    ```json
    {
      "cart": {
        "email": "customer@example.com"
      }
    }
    ```

    The email is required for order notifications and account management.

    **Add a shipping address**

    ```bash
    POST https://myco.fluid.app/api/carts/{cart_token}/address
    ```

    ```json
    {
      "address": {
        "first_name": "Jane",
        "last_name": "Doe",
        "address1": "123 Main St",
        "city": "Austin",
        "state": "TX",
        "zip": "78701",
        "country": "US"
      }
    }
    ```

    Setting a shipping address triggers automatic shipping option calculation (see Step 5).

  </Step>

  <Step title="Get Shipping Options">

    You do not need to call a separate endpoint to retrieve shipping options. Once you add a shipping address, the cart response automatically includes an `available_shipping_options` array on every subsequent retrieve or update call.

    Options are calculated based on:

    - Destination country and region
    - Cart contents and weight
    - Customer type (retail vs. wholesale)
    - Any special handling requirements

    The options update automatically whenever you change the shipping address or modify cart contents that affect shipping calculations. Retrieve the current cart to read the latest options:

    ```bash
    GET https://myco.fluid.app/api/carts/{cart_token}
    ```

    Review the `available_shipping_options` array in the response and choose a `shipping_option_id` for the next step.

  </Step>

  <Step title="Select Shipping Method">

    Apply the chosen shipping option to the cart. This updates the cart totals to reflect the shipping cost.

    <CodeGroup>

    ```bash cURL
    curl -X PATCH "https://myco.fluid.app/api/carts/CT-SECURE_TOKEN_XYZ/shipping_method" \
      -H "Content-Type: application/json" \
      -d '{
        "cart": {
          "shipping_option_id": "ship_standard_us"
        }
      }'
    ```

    ```javascript JavaScript
    await fetch(
      'https://myco.fluid.app/api/carts/CT-SECURE_TOKEN_XYZ/shipping_method',
      {
        method: 'PATCH',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          cart: { shipping_option_id: 'ship_standard_us' },
        }),
      }
    )
    ```

    </CodeGroup>

    After setting the shipping method, retrieve the cart again to confirm the updated order total.

  </Step>

  <Step title="Add Payment">

    Payment collection for headless integrations is handled outside the Cart API. Fluid supports the following payment methods at checkout:

    - **Hosted checkout** — redirect the customer to Fluid's hosted payment page
    - **Vaulted cards** — use previously saved cards via the customer payment-method endpoints in the Checkout v2026-04 API
    - **Apple Pay** — digital wallet support routed through your configured PSP
    - **Google Pay** — digital wallet support routed through your configured PSP

    For applications that need to vault cards programmatically (without redirecting to hosted checkout), refer to the customer payment-method endpoints in the [Checkout v2026-04 API Reference](/docs/openapi/checkout-v2026-04).

  </Step>

  <Step title="Complete Checkout">

    Submit the final POST to convert the cart into an immutable order. The routing engine validates all required fields, confirms product availability, validates the payment method, and applies any business rules before processing.

    <CodeGroup>

    ```bash cURL
    curl -X POST "https://myco.fluid.app/api/checkout" \
      -H "Content-Type: application/json" \
      -d '{
        "cart_token": "CT-SECURE_TOKEN_XYZ"
      }'
    ```

    ```javascript JavaScript
    const response = await fetch('https://myco.fluid.app/api/checkout', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ cart_token: 'CT-SECURE_TOKEN_XYZ' }),
    })
    const { order } = await response.json()
    console.log(order.id) // Your new order ID
    ```

    </CodeGroup>

    A successful response includes the new order object. If validation fails, you receive a specific error response describing what to correct before retrying.

    <Warning>
      The cart token becomes permanently invalid after a successful checkout. Do not attempt to reuse it. Use the returned order ID to track post-purchase status via the [Order API Reference](/docs/openapi/admin-v0).
    </Warning>

    After checkout completes, your implementation may also trigger:

    - Order confirmation emails
    - Commission calculations for distributors
    - Inventory updates
    - Shipping label generation

    Each of these features has dedicated API endpoints documented in the [Admin API Reference](/docs/openapi/admin-v0).

  </Step>

</Steps>
