The FairShare SDK gives you a complete suite of commerce components and JavaScript APIs through a single `<script>` tag. Because it ships as a standalone bundle with automatic style injection, you can drop it into any HTML page without a build pipeline, a bundler, or a framework dependency. Whether you're running a vanilla HTML site, a WordPress blog, or a React application, the SDK works the same way.

## Quick Start

Add the following script tag to your page, then place components anywhere in your HTML:

```html
<script
  id="fluid-cdn-script"
  src="https://assets.fluid.app/scripts/fluid-sdk/latest/web-widgets/index.js"
  data-fluid-shop="your-shop-id"
></script>
```

Once the script is on the page, add components wherever you need them:

```html
<fluid-media library-id="your-library-id"></fluid-media>
<fluid-attribution></fluid-attribution>
```

That's all the setup required. The SDK initializes automatically, fetches your shop configuration, and handles attribution in the background.

### Components

| Component | Description |
|---|---|
| Media | Display media content with popover or inline modes |
| Playlist | Display a collection of media items |
| Lead Capture | Capture customer leads with configurable forms |
| Attribution | Display affiliate attribution information |
| Country | Country selector for locale settings |
| User | Display authenticated user info and login/logout |
| Language | Language selector for locale settings |

### JavaScript API

| API Group | Description |
|---|---|
| Cart Functions | Add items, update quantities, open/close the drawer, and check out |
| Settings & Configuration | Manage locale, language, country, and shop configuration |
| Events & Analytics | Track custom events, flush buffers, and retrieve attribution data |

### Declarative HTML Controls

Use data attributes for the most common cart actions without writing a single line of JavaScript:

```html
<!-- Open and close the cart drawer -->
<button data-fluid-cart="open">View Cart</button>
<button data-fluid-cart="close">Close Cart</button>

<!-- Add a product to cart -->
<button data-fluid-add-to-cart="123456">Add to Cart</button>
```

## How Configuration Works

The SDK resolves settings through a three-layer priority system. Higher layers override lower ones:

<Steps>
  <Step title="API Settings (lowest priority)">
    When the SDK initializes, it fetches your shop's default configuration—language, country, currency, and other defaults—from the Fluid API.
  </Step>
  <Step title="Script Tag Attributes">
    Attributes on the `<script>` tag such as `data-fluid-shop`, `data-share-guid`, and `data-pusher-enabled` override the API defaults for every page load.
  </Step>
  <Step title="Your UI Code (highest priority)">
    JavaScript calls made at runtime take the highest priority and overwrite any previously resolved value. Note that updates via code **replace** the existing value rather than merging with it.

    ```javascript
    // This overrides whatever the API returned
    await window.FairShareSDK.updateLocaleSettings({
      language: "es",
      country: "MX",
    });
    ```
  </Step>
</Steps>

## Key Benefits

- **No build process required** — Works with any HTML page, no bundler needed
- **Automatic style injection** — No separate CSS file to include or manage
- **Server-side attribution** — Attribution is calculated automatically from the page URL
- **Framework agnostic** — Works equally well with React, Vue, vanilla JS, or any other framework
